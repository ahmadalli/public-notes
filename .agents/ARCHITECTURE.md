# Architecture and Development

Development-focused reference for this Docusaurus site. For content editing rules, see `.agents/CONTENT.md`. For Docusaurus upgrades, see `.agents/UPGRADE.md`.

## Project Structure

```
docs/           Content (Markdown/MDX), organized by numbered directories
blog/           Blog posts (weekly changelogs)
src/
  components/   Custom React components (ChildPages, WikimediaToMarkdown)
  css/          Global CSS overrides (Infima theme variables)
  pages/        Standalone pages
  theme/        Docusaurus theme component overrides (MDXComponents)
static/         Static assets (images, robots.txt)
tools/          Utility scripts (changelog generation)
.vale/          Vale prose linter styles and vocabularies
```

## Commands

```bash
npm ci                     # Install dependencies
npm start                  # Dev server (runs: docusaurus start --no-open)
npm run build              # Production build — THIS IS THE MAIN VALIDATION STEP
npm run typecheck          # TypeScript type checking (runs: tsc)
vale docs/                 # Prose linting (Vale must be installed: https://vale.sh)
```

- There is **no test runner** (no Jest, Vitest, etc.) and **no ESLint/Prettier**.
- The primary validation is `npm run build`. It fails on broken links.
- Use `npm`, not yarn or pnpm. A `package-lock.json` is present.

## CI/CD

Two GitHub Actions workflows (both Node 20):

- **`deploy.yml`**: pushes to `main` run `npm ci && npm run build`, then deploy to GitHub Pages.
- **`test-deploy.yml`**: PRs to `main` run `npm ci && npm run build` (build only, no deploy).
- Full git history is fetched (`fetch-depth: 0`) because Docusaurus uses it for "last updated" timestamps.

## Code Style

### TypeScript / TSX

- **Indentation**: 2 spaces (enforced by `.editorconfig`)
- **Strings**: Double quotes
- **Semicolons**: Yes
- **Types**: Use `type` (not `interface`)
- **Components**: `React.FC<Props>` with `export default`
- **Naming**: PascalCase for components/types, camelCase for functions/variables
- **Imports**: ES modules; use `@site/` alias for project-root imports
- **Variables**: `const`/`let`, never `var`

Example of a correct component:

```tsx
import React from "react";

type ChildPagesProps = {
  depth?: number;
};

const ChildPages: React.FC<ChildPagesProps> = ({ depth = 1 }) => {
  return <div>...</div>;
};

export default ChildPages;
```

### JavaScript (Theme Overrides)

- Files in `src/theme/` import from `@theme-original/` and re-export with additions.
- Add comments explaining what each override does.

### CSS

- Use CSS custom properties (Infima variables) in `:root` and `[data-theme='dark']`.
- Keep custom CSS minimal. Prefer Docusaurus/Infima defaults.

## Docusaurus Configuration

Key settings in `docusaurus.config.ts`:

| Setting | Value | Effect |
|---------|-------|--------|
| `onBrokenLinks` | `"throw"` | Broken internal links **fail the build** |
| `trailingSlash` | `true` | All URLs end with `/` |
| `docs.routeBasePath` | `"/"` | Docs are at site root |
| `blog.routeBasePath` | `/blog` | Blog at `/blog` |
| Sidebar | `type: "autogenerated"` | Auto-generated from filesystem |
| Search | `docusaurus-lunr-search` | Client-side search |
| Syntax highlighting | Prism: GitHub (light) / Dracula (dark) | Extra languages: Python, Perl |
| Default color mode | `dark` | -- |

## Common Development Tasks

### Adding a New Component

1. Create `src/components/my-component.tsx`.
2. Use `type` for props, `React.FC<Props>` pattern, `export default`.
3. Import in MDX: `import MyComponent from '@site/src/components/my-component'`.
4. Run `npm run typecheck` and `npm run build`.

### Modifying Theme

1. Swizzle or manually override in `src/theme/`.
2. Import from `@theme-original/` and extend.
3. Run `npm run build` to verify.

## Vale Prose Linting

Vale is configured in `.vale.ini` with `MinAlertLevel = suggestion`:

- **Microsoft Writing Style Guide** rules enabled (disabled: `Headings`, `FirstPerson`, `We`, `ComplexWords`, `Wordiness`, `HeadingAcronyms`, `HeadingPunctuation`).
- **Custom rule**: `PublicNotes/HeadingsStyle` enforces AP title-style headings.
- **Vocabulary**: `.vale/config/vocabularies/PublicNotes/accept.txt` has accepted terms. Add new terms here if Vale flags legitimate words.
- MDX files are treated as Markdown (`mdx = md` in config).
